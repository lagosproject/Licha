# Security Policy

We take the security of **Licha (TwitchChatTTS)** seriously. If you believe you have found a security vulnerability, please read this policy to learn how to report it responsibly.

## Supported Versions

Only the latest release of Licha is actively supported with security updates. We recommend always running the latest version available on the Google Play Store or built from the main branch.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.0.10 | :white_check_mark: |
| < 0.0.10  | :x:                |

## Reporting a Vulnerability

**Do not open a public issue on GitHub for security vulnerabilities.**

Instead, please report security vulnerabilities by emailing the maintainers at:
`security@lakescorp.com` (replace with your secure contact email or project contact)

Please include the following details in your report:
- **Type of vulnerability** (e.g., credential leakage, insecure storage, arbitrary code execution).
- **Steps to reproduce** the vulnerability (with code or screenshots if possible).
- **Potential impact** of the vulnerability.
- **Any suggested fix** or mitigation, if known.

We will acknowledge receipt of your report within **48 hours** and provide a timeline for resolution and disclosure.

## Our Commitment

We commit to:
1. Responding promptly and investigating the report.
2. Keeping you informed of our progress.
3. Giving credit to vulnerability reporters in our security advisories (if desired).
4. Not taking legal action against security researchers who comply with this policy in good faith.
